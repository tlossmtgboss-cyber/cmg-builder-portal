Seven ways to build the same thing.

Same Builder Packet — Builder Acknowledgment, Builder & Project Review, Texas Supplement, Draw Request, W9. Same conditional logic for TX + Draw + HOA. Same Smart Docs portal. Seven completely different ways for the builder to fill it out.

New · Submission Flow

When the builder hits Submit, the system now assembles the five filled PDFs (BAF, BPR, TX Supplement, Draw Request, W9) into a single packet and emails it to the loan officer on file. The builder sees a confirmation showing exactly what the LO received. Live in Variants 5, 6, and 7.

Variant 06

AutoPilot

Drop docs · AI extracts

Builder drops W9, license, COI, construction contract. AutoPilot OCR's everything and pre-fills 78% of the application. Confidence scores per field. Flags issues (e.g., "CMG not listed as Cert Holder"). Builder just confirms.

  • Best for: builders who keep docs in order
  • 4 extraction cards: W9 (97%), License (99%), GL Insurance (84%), Contract (94%)
  • Auto-detects TX property + calculates draw ceiling
  • Submit → assembles packet → emails LO

What's identical across all seven

  • Every PDF field from the Builder Packet — 90+ fields across 12 sections (BAF, BPR, Texas Supplement, Draw Request, W9)
  • Same conditional logic: Texas property unlocks Section 10 + Texas confirmation doc. Draw at closing unlocks Section 11 (max 10% of contract less borrower deposit). HOA / PUD adds HOA Approval to Smart Docs.
  • Same Smart Docs portal: 12 standing documents + 2 conditional (HOA, TX confirmation). Auto-validation for type, age, and quality.
  • Same theme: #FAF7F1 cream · #1F3D2E forest green · #B8924A gold · Fraunces + Geist + Geist Mono (V7 adds Crimson Pro for paper feel)
  • Same repeater groups: 1+ Principals, 3 Trade refs, 3 Suppliers (one marked Primary Lumber), 1 required + 1 optional Banking ref
